# [Android App] 설치 및 실행 가이드

본 문서는 안드로이드 클라이언트(Client)의 설치 및 실행 방법을 설명합니다.
테스트 목적에 따라 **① 제공된 APK로 즉시 실행**(로컬 서버 실행 시 동작하지 않음)하거나, **② 소스 코드를 빌드하여 로컬 서버와 연동**할 수 있습니다.

---

## 1. 시제품 APK 바로 실행
제출된 APK 파일은 **이미 배포된 클라우드 서버(EC2)**와 통신하도록 설정되어 있습니다.

1. **설치 파일:** `Client/lucie-coach.apk`
2. **설치 방법:**
   * **에뮬레이터:** 실행된 에뮬레이터 화면에 APK 파일을 드래그 앤 드롭(Drag & Drop)하여 설치
   * **실물 기기:** USB 저장소 또는 드라이브로부터 다운로드하고, 다운로드된 apk 파일을 기기 저장소에서 찾아('내 파일'에서 검색) 터치하면 설치됩니다. 이때 '알 수 없는 출처'를 허용해야 합니다.
3. **실행:** 별도의 서버 설정 없이 로그인 및 기능 테스트가 가능합니다.

---

## 2. 로컬 서버 연동 실행 (소스 코드 빌드)
로컬 PC에서 서버(`localhost`)를 직접 구동하는 방법입니다.

### 2-1. 사전 준비
1. **서버 실행:** [Server] 폴더의 가이드를 따라 로컬 서버를 먼저 실행해주세요.
2. **소스 코드 준비:**
   * 깃허브 저장소 `https://github.com/2025-eStart/Lucie` 에서 `Source_Code/android-app` 폴더를 다운로드하여 Android Studio로 엽니다.

### 2-2. 서버 주소 변경 (필수)
로컬 서버와 연결하기 위해 앱 소스 코드 내의 **API 기본 주소(BASE_URL)**를 수정해야 합니다.

1. **수정할 파일 경로:**
   `android-app/app/build.gradle.kts`에서 아래 코드를 검색합니다.(키워드: '로컬 호스트 사용 시')
   `buildConfigField`에서 "http://10.0.2.2:8123"를 로컬 호스트 주소로 수정해야 합니다.
   수정 후 `buildConfigField` 블럭은 하나만 남기고 주석처리합니다.
   ```kotlin
   // 로컬 호스트 사용 시: 애뮬레이터용 주소
   // buildConfigField("String", "API_BASE_URL", "\"http://10.0.2.2:8123/\"")
   ```
   
2. **주소 수정 가이드:**
   실행 환경에 따라 아래 주소 중 하나를 선택하여 `BASE_URL` 값을 변경하세요.

   **[CASE A] Android Studio 에뮬레이터 사용 시 (추천)**
   에뮬레이터에서 `10.0.2.2`는 실행 중인 PC(Host)를 가리킵니다.
   ```kotlin
   // 예시 코드
   const val BASE_URL = "[http://10.0.2.2:8000/](http://10.0.2.2:8000/)"

   ** [CASE B]** **실물 스마트폰** 사용 시 PC와 폰이 **동일한 Wi-Fi**에 연결되어 있어야 합니다. PC의 IP 주소를 확인하여 입력하세요.
    * PC IP 확인법: 터미널에 ipconfig(Windows) 또는 ifconfig(Mac) 입력
    ```kotlin
    // 예시 코드 (PC의 IP가 192.168.0.5 인 경우)
    const val BASE_URL = "[http://192.168.0.5:8000/](http://192.168.0.5:8000/)"
    ```

### 2-3. 빌드 및 실행
1. Android Studio 우측 상단의 Run (▶) 버튼을 눌러 앱을 빌드하고 실행합니다.
2. 로컬 서버 터미널에 요청 로그가 올라오는지 확인합니다.
